How I work

Approach to tasks

  • Decompose large tasks into small ones with explicit acceptance criteria; validate assumptions as early as possible.
  • Read existing code and look for established patterns in the repo before introducing new abstractions.
  • Small commits with clear messages; large changes split into a chain of focused PRs.
  • For tricky cases — sketch the data flow / sequence diagram first (Figma / Excalidraw), then code.
  • Code review focused on correctness and business logic, not just style.

AI in everyday work

  • tools Claude Code, Cursor, ChatGPT — depending on the task: refactors and routine in IDE, research / debugging in chat.
  • how Treat AI as a drafting partner: boilerplate, types, tests, exploring unfamiliar APIs, parsing stack traces.
  • limits Never delegate understanding — final decision and review are mine, AI output is verified by types and tests.
  • prompts Iterative prompting with explicit input, output and constraints; small step, check, refine.
  • legacy On legacy code AI helps to quickly map the architecture and produce the first diff for review.
